내돈내픽 (Naedonnaepick)
내돈내픽은 머신러닝을 활용하여 가격대, 위치, 개인 취향을 종합적으로 고려한 맞춤형 음식점 추천 모바일 앱입니다. 사용자가 설정한 예산과 선호 태그(맛/분위기 등)를 기반으로 주변 음식점을 추천해 주고, 식사 예산 관리와 실시간 지역 채팅 기능을 제공합니다. 이 프로젝트는 React Native 모바일 앱(프론트엔드)과 Spring Boot 서버(백엔드)로 구성된 **팀 프로젝트(4명)**이며, 작성자는 팀장으로서 핵심 프론트엔드 구현 및 채팅 기능과 음식점 추천 기능을 담당하였습니다.
기술 스택 (Tech Stack)
Frontend: React Native (Expo) + JavaScript/TypeScript, 상태 관리로 Zustand 사용, SockJS/Stomp를 통한 WebSocket 통신 지원.
Backend: Spring Boot (Spring MVC, Spring Data JPA) 기반 REST API 서버 - MySQL 연동 및 JPA로 데이터 관리.
Database: MySQL (AWS RDS에 배포)를 사용하여 회원, 음식점, 채팅, 예산 등의 데이터를 저장.
ML/AI: Python Flask 서버의 ML 모델들과 연동 – 랜덤 포레스트 기반 추천 알고리즘과 한국어 감성 분석 모델(KcELECTRA-base) 활용.
Infra: AWS EC2 서버에서 백엔드와 Flask ML 서버 구동, 이미지/영상 저장 등. Selenium WebDriver로 공공데이터 크롤링 및 초기 데이터 수집.

주요 기능 (Key Features)
1. 맞춤 음식점 추천 (Personalized Restaurant Recommendations)
사용자의 현재 위치 또는 선택 지역, 희망 가격 범위(예산), 그리고 선호 태그(맛/분위기 등 선호도 점수)를 입력받아 AI 모델이 최적의 음식점을 추천합니다. 추천 로직은 미리 학습된 머신러닝 모델(랜덤 포레스트)을 통해 구현되었으며, Spring Boot 서버가 해당 입력을 Flask 기반 추천 모델에 요청하여 결과를 받아옵니다. 앱 화면 상단에 필터(가격, 거리, 취향)를 설정하고 “추천” 버튼을 누르면, 조건에 맞는 음식점 리스트가 예측 확률이 높은 순으로 정렬되어 표시됩니다 (동일 확률일 경우 거리순). 각 추천 항목에는 음식점 이름, 예상 선호도(매칭 확률), 거리 등의 정보가 제공되어 사용자가 한눈에 비교할 수 있습니다. 사용자가 마음에 드는 음식점을 선택하고 “먹기” 버튼을 누르면 해당 메뉴 가격이 예산에서 차감되고 소비 내역에 기록됩니다.
2. 근처 음식점 확인 (View Nearby Restaurants)
앱은 사용자의 GPS 기반 현재 위치를 활용하여 주변의 음식점을 보여주는 기능도 제공합니다. 홈 화면에서 “내 주변 음식점” 목록을 보면 현재 위치를 중심으로 거리순 정렬된 음식점들을 확인할 수 있습니다. 사용자가 위치 권한을 허용하면, 현 위치 좌표를 기반으로 일정 반경 내의 식당들을 검색하고 목록으로 표시합니다. 이를 통해 사용자는 별도 검색 없이도 가까운 식당을 손쉽게 찾아볼 수 있습니다. (추후 지도를 연동하여 시각화하는 것도 고려할 수 있습니다.)
3. 예산 관리 및 소비 기록 (Budget Management & History)
월별 예산을 설정하고 식비 지출을 관리할 수 있는 기능입니다. 사용자는 앱 내 예산 관리 화면에서 이번 달 총 예산 금액을 입력할 수 있고, 앱은 **현재 날짜 기준으로 사용 가능 금액(남은 예산)**을 자동 계산하여 보여줍니다. 음식점을 방문하여 결제를 했다면 해당 금액을 기록할 수 있으며, 앞서 언급한 “먹기” 버튼을 통해 추천 리스트에서 바로 소비를 등록하면 예산이 차감됩니다. 예산 화면에는 최근 소비 내역 (마지막 3건 등)도 요약되어 표시되어 사용자가 지출 내역을 한눈에 파악할 수 있습니다. 상세 내역 확인을 위해 별도의 모달 또는 화면에서 지금까지의 모든 식비 지출 기록을 스크롤하여 볼 수도 있습니다. 이를 통해 사용자는 예산 대비 지출 현황을 쉽게 모니터링하고, 과도한 지출을 피하면서 계획적으로 식비를 관리할 수 있습니다.
4. 메뉴 가격 필터링 (Affordable Menu Highlight)
한 음식점의 상세 페이지나 추천 목록에서 해당 예산으로 구매 가능한 메뉴를 확인할 수 있는 기능입니다. 이를 구현하기 위해 네이버 지도에서 음식점별 메뉴 데이터를 크롤링하여 DB에 저장하였습니다. 약 성남시 내 9,300여 개 음식점 중 메뉴가 공개된 4,600여 개 음식점의 메뉴명, 가격 정보를 수집하여 총 약 6만여 개의 메뉴 데이터를 확보하였고, 이러한 데이터는 MySQL에 연동되어 있습니다. 앱에서는 사용자의 남은 예산과 각 음식점의 메뉴 가격을 비교하여, 예산 범위 이내의 메뉴에는 특별 표시를 하거나 리스트 상단에 별도로 보여줍니다. 이로써 사용자는 음식점 목록을 탐색할 때 내 예산으로 먹을 수 있는 메뉴가 있는지 바로 확인할 수 있어, 메뉴 선택에 소요되는 시간을 줄여줍니다.
5. 지역 기반 채팅 (Location-Based Chat Rooms)
사용자들 간에 실시간 소통이 가능하도록, 지역별로 구분된 채팅방 기능을 제공합니다. 예를 들어 성남시 분당구 지역 채팅방에 들어가면 그 지역에 있는 다른 사용자들과 맛집 정보나 후기를 자유롭게 공유할 수 있습니다. 프론트엔드에서는 SockJS를 사용하여 WebSocket 연결을 관리하고, 백엔드에서는 Spring WebSocket(STOMP)를 통해 메시지 브로커를 구성하여 실시간 채팅을 구현하였습니다. 다수의 사용자가 동시에 참여해도 원활히 동작하도록 여러 채팅방 개설, 동시 입장, 메시지 브로드캐스트 등이 구현되어 있습니다. 주고받은 채팅 메시지는 모두 DB에 영구 저장되며, 채팅방 입장 시 과거 대화내역을 불러와 표시해줍니다. 이를 통해 새로운 사용자도 이전 대화 내용을 확인할 수 있고, 중요 정보가 누락되지 않습니다. 이 기능은 지역 커뮤니티 활성화와 사용자 간 정보 교류를 목적으로 하며, 맛집 후기나 추천을 실시간으로 얘기할 수 있는 공간을 제공합니다.
6. 채팅 투표 이벤트 (Chat Poll Events)
채팅방 내에서는 투표 기능을 통해 사용자 참여형 이벤트를 진행할 수 있습니다. 예를 들어 "이번 주말에 어디서 모임을 가질까요?"와 같은 주제로 투표를 생성하면 채팅방의 참여자들이 실시간으로 투표하고 결과를 공유할 수 있습니다. 이러한 투표 이벤트는 사용자 간의 재미 요소를 더해줄 뿐만 아니라, 집단 지성을 활용해 최고의 맛집을 선정하는 등의 용도로 활용됩니다. (추가로, 채팅 메시지에 대한 감성 분석도 시도하여 긍정/부정 여부를 파악하고 이를 투표와 연계하는 계획이 있었으며, BERT 기반 모델로 구현을 모색하였습니다.) 투표 결과는 실시간으로 갱신되어 모두에게 공유되고, 투표가 완료되면 최다 득표 옵션을 하이라이트하여 보여줍니다. 이 기능을 통해 사용자들은 다양한 의견을 수렴하거나 재미있는 이벤트를 진행할 수 있습니다.
7. 간편한 회원 인증 (User Authentication)
다양하고 간편한 회원 가입/로그인 기능을 지원하여 신규 사용자의 진입 장벽을 낮추었습니다. 일반적인 이메일 회원가입 및 로그인 외에 소셜 로그인(카카오 등)을 통합하여 원클릭으로 가입할 수 있도록 구현했습니다. 회원 가입 시 입력 실수나 중복 확인 등의 예외 처리를 철저히 수행하여 안정적인 회원 관리가 가능하며, 로그인 상태 유지와 전역 사용자 정보 관리는 Zustand로 상태 관리하여 앱 전반에서 활용합니다. 또한 로그아웃 기능과, 비밀번호 찾기/재설정, 프로필 편집 등 기본적인 계정 관리 기능도 포함되어 있습니다. 권한이 필요한 기능(예: 채팅 참여, 예산 기록 등)에 접근할 때는 JWT 기반 토큰 인증으로 백엔드 검증을 거치도록 설계하여 보안도 신경 썼습니다. 전반적으로 직관적인 UI 흐름을 통해 회원 인증 과정을 간소화하고, 다양한 방법으로 사용자가 편리하게 서비스를 이용할 수 있게 하였습니다.

실행 방법 (How to Run)
로컬 환경에서 내돈내픽 앱을 실행하려면 다음 단계를 따르면 됩니다:
백엔드(Spring Boot) 서버 실행:
MySQL 데이터베이스를 준비하고 프로젝트의 application.yml 또는 application.properties 파일에 DB 접속 정보(URL, username, password)를 설정합니다. (기본 스키마와 필요한 테이블은 JPA 엔티티 기반으로 자동 생성되며, 초기 데이터는 제공된 SQL 스크립트 또는 크롤링 모듈을 통해 주입 가능합니다.)
Java 11+ 환경에서 Spring Boot 애플리케이션을 빌드 및 실행합니다. 예를 들어 Gradle 사용 시 ./gradlew bootRun 또는 java -jar backend/build/libs/naedonnaepick.jar 명령으로 서버를 기동합니다.
서버가 실행되면 http://localhost:8080에서 REST API가 활성화됩니다 (기본 포트 8080). WebSocket을 통한 채팅 기능도 /ws/chat 엔드포인트로 설정됩니다.

Node.js 및 Expo CLI가 설치된 환경이 필요합니다. 레포지토리의 frontend/ 디렉토리로 이동하여 의존성을 설치합니다:

앱 테스트:
앱이 기기에 로드되면, 회원가입 등 기본 시나리오를 따라가며 기능을 테스트합니다. 백엔드와 연결되어 추천 목록 가져오기, 예산 등록/차감, 채팅 메시지 송수신 등이 정상적으로 작동하는지 확인합니다.
채팅 기능을 테스트하려면 두 개 이상의 기기 또는 Emulator에서 서로 같은 지역 채팅방에 접속하여 메시지를 주고받아 볼 수 있습니다. (백엔드 WebSocket 로그에서도 구독/메시지 전송 상황을 확인 가능)
참고: 처음 실행 시 크롤링된 음식점/메뉴 데이터와 ML 모델 파일이 필요합니다. 해당 데이터셋 및 모델은 용량 문제로 별도 제공되며, backend/resources/ 경로 등에 모델 파일을 넣고 경로를 맞춰주어야 추천 기능이 온전히 동작합니다. 모델 서버(Flask)가 분리되어 있는 경우, Spring Boot 설정에서 Flask API URL을 지정하고 Flask 서버도 함께 실행해야 추천 결과를 얻을 수 있습니다. 기본적인 추천/채팅/예산 기능 자체는 모델 서버 없이도 더미 데이터로 테스트 가능하도록 구성되어 있습니다.
참고 및 향후 계획
내돈내픽 프로젝트는 한국폴리텍대학 성남캠퍼스 인공지능소프트웨어과 팀 프로젝트로 수행되었으며, 제한된 기간 내에 핵심 기능들을 구현하였습니다. 향후에는 다음과 같은 보완 및 발전 방향을 고려하고 있습니다:
UI/UX 개선: 사용자의 피드백을 반영하여 화면 디자인을 개선하고, 지도 뷰를 활용한 주변 식당 시각화, 알림 기능 등을 추가.
기능 고도화: 채팅 메시지의 감성 분석 결과를 바탕으로 한 자동 태그 생성이나 맞춤 이벤트 알림, 즐겨찾기한 음식점 관리 등의 기능 확대.
성능 및 안정성: 대규모 사용자 트래픽을 대비한 성능 튜닝, 채팅 서버 확장 (예: STOMP 대신 Kafka 도입 검토) 등 안정적인 서비스 운영을 위한 개선.
끝으로, 내돈내픽은 가격에 민감한 사용자들이 한정된 예산 안에서 최고의 선택을 할 수 있도록 도와주는 유용한 앱입니다. 또한 지역 사회의 사용자들이 실시간으로 소통하며 정보를 나눌 수 있는 장을 제공함으로써, 단순한 추천 엔진을 넘어 커뮤니티 기반의 서비스로 발전할 가능성을 열어두었습니다. 이 프로젝트를 통해 팀원 모두가 기획부터 배포까지 앱 개발 전 과정을 경험하였고, 특히 머신러닝 모델과 모바일 앱, 웹소켓 서버를 통합하는 값진 실무 경험을 쌓을 수 있었습니다. 앞으로도 해당 프로젝트를 개선시키며 배운 점을 바탕으로 더욱 발전된 서비스를 만들어 나갈 계획입니다.
